# Quickstart

This guide gets you from zero to a working MX8 loader in under 2 minutes.

## Prerequisites

- Python 3.8 or higher
- pip installed

## Install MX8

<Steps>
  <Step title="Install the package">
    ```bash
    pip install mx8
    ```
  </Step>

  <Step title="Verify installation">
    ```bash
    python -c "import mx8; print(mx8.__version__)"
    ```
  </Step>
</Steps>

## Your First Loader

Create a file called `first_loader.py`:

```python
import mx8

loader = mx8.load(
    "s3://your-bucket/dataset/",
    batch_size_samples=64,
    max_ram_gb=12,
    profile="balanced",
)

print("manifest_hash:", loader.manifest_hash)

total = 0
for batch in loader:
    total += len(batch.sample_ids)
    print(f"Delivered {total} samples so far")

print(f"Total samples: {total}")
```

<Note>
  Replace `s3://your-bucket/dataset/` with your actual S3 path or local directory.
</Note>

Run it:

```bash
python first_loader.py
```

## What Just Happened?

MX8 automatically:

1. **Indexed your dataset** - Scanned the S3 prefix and created a manifest
2. **Computed a snapshot hash** - Pinned the dataset state for deterministic replay
3. **Enforced memory bounds** - Limited pipeline memory to 12GB
4. **Delivered batches** - Fetched, decoded, and packed samples with bounded queues

The manifest is cached at `~/.mx8/manifests/` for future runs.

## Training Example

Here's a minimal training loop using PyTorch:

```python
import mx8
import torch
import torch.nn.functional as F

loader = mx8.load(
    "s3://your-bucket/dataset/",
    batch_size_samples=32,
    max_ram_gb=4,
    profile="balanced",
)

model = torch.nn.Linear(1024, 10)  # example model
opt = torch.optim.SGD(model.parameters(), lr=0.01)

for batch in loader:
    payload_u8, offsets_i64, sample_ids_i64 = batch.to_torch()
    
    # Your training logic here
    # This example assumes fixed-size samples of 1024 bytes
    x = payload_u8.reshape(-1, 1024).float() / 255.0
    y = (sample_ids_i64 % 10)  # dummy labels
    
    pred = model(x)
    loss = F.cross_entropy(pred, y)
    
    opt.zero_grad()
    loss.backward()
    opt.step()
    
    print(f"Loss: {loss.item():.4f}")
```

<Warning>
  This example assumes all samples are exactly 1024 bytes. For variable-length data, use `offsets` to extract individual samples.
</Warning>

## Image Training Example

For image datasets, use the `mx8.image` loader:

```python
import mx8
import torch
import torch.nn.functional as F

loader = mx8.image(
    "s3://your-bucket/images/",
    batch_size_samples=64,
    resize_hw=(224, 224),
    augment="imagenet",  # automatic augmentation + normalization
    max_ram_gb=8,
    profile="balanced",
)

print("Classes:", loader.classes)  # ['cat', 'dog', ...] or None

model = torch.nn.Sequential(
    torch.nn.Flatten(),
    torch.nn.Linear(224 * 224 * 3, 10)
)
opt = torch.optim.SGD(model.parameters(), lr=0.01)

for images, labels in loader:
    # images: [B, C, H, W] float32 tensor
    # labels: [B] int64 tensor
    
    logits = model(images)
    loss = F.cross_entropy(logits, labels)
    
    opt.zero_grad()
    loss.backward()
    opt.step()
    
    print(f"Loss: {loss.item():.4f}")
```

## Multi-Epoch Training

Loaders are single-pass. For multi-epoch training, create a new loader each epoch:

```python
import mx8

# Pin the snapshot once
manifest_hash = mx8.resolve_manifest_hash("s3://bucket/train@refresh")
pinned = f"s3://bucket/train@sha256:{manifest_hash}"

for epoch in range(10):
    loader = mx8.load(
        pinned,
        batch_size_samples=64,
        max_ram_gb=8,
        profile="balanced",
    )
    
    for batch in loader:
        train_step(batch)
    
    print(f"Epoch {epoch} complete")
```

<Note>
  Pinning the manifest hash ensures every epoch sees the exact same dataset snapshot.
</Note>

## Monitoring Pipeline Health

Check loader stats to monitor throughput and memory:

```python
import mx8

loader = mx8.load("s3://bucket/dataset/", max_ram_gb=12)

for batch in loader:
    train_step(batch)
    
    # Print human-readable stats
    if batch_num % 100 == 0:
        print(mx8.stats(loader))
    
    # Or access raw metrics
    stats = loader.stats()
    if stats["ram_high_water_bytes"] > 10 * 1024**3:  # 10GB
        print("Warning: approaching memory limit")
```

Key stats fields:
- `delivered_samples_total` - Total samples delivered so far
- `process_rss_bytes` - Current process memory usage
- `ram_high_water_bytes` - Peak pipeline memory usage
- `elapsed_seconds` - Time since loader start

## Distributed Training

For multi-node training, use the coordinator context manager:

```python
import os
import mx8

rank = int(os.environ["RANK"])
world_size = int(os.environ["WORLD_SIZE"])

# Start coordinator on rank 0, all ranks wait automatically
with mx8.coordinator(rank=rank, world_size=world_size) as coord:
    loader = mx8.load(
        "s3://bucket/train/",
        batch_size_samples=512,
        max_ram_gb=24,
        job_id="train-001",
        cluster_url=coord.url,
    )
    
    for batch in loader:
        train_step(batch)
```

Launch with torchrun:

```bash
torchrun --nproc_per_node=8 train.py
```

## Common Patterns

### Pattern: Checkpoint and Resume

```python
loader = mx8.load("s3://bucket/dataset/", max_ram_gb=12)

for batch in loader:
    train_step(batch)
    
    if should_checkpoint():
        token = loader.checkpoint()
        save_checkpoint(model, token)

# Resume from checkpoint
loader = mx8.load(
    "s3://bucket/dataset/",
    max_ram_gb=12,
    resume_from=token,
)
```

### Pattern: Specific Sample Range

```python
# Load only samples 1000-2000
loader = mx8.load(
    "s3://bucket/dataset/",
    start_id=1000,
    end_id=2000,
    batch_size_samples=64,
    max_ram_gb=8,
)
```

### Pattern: Auto-packing on First Run

```python
# Automatically pack a raw S3 prefix on first run
loader = mx8.load(
    "s3://bucket/raw-images/",
    autopack=True,
    autopack_shard_mb=512,
    max_ram_gb=12,
)
```

The packed manifest and shards are written to the same prefix. Subsequent runs skip packing automatically.

## Troubleshooting

### Error: "max_ram_gb cap would be breached"

Increase `max_ram_gb` or reduce `batch_size_samples`:

```python
loader = mx8.load(
    "s3://bucket/dataset/",
    batch_size_samples=32,  # reduced from 64
    max_ram_gb=16,          # increased from 12
)
```

### Error: "manifest not found"

For local development, provide an explicit manifest path:

```python
loader = mx8.load(
    "/tmp/dev@refresh",
    manifest_path="/path/to/manifest.tsv",
    max_ram_gb=8,
)
```

### Performance: Slow throughput

Try the `throughput` profile:

```python
loader = mx8.load(
    "s3://bucket/dataset/",
    max_ram_gb=24,
    profile="throughput",  # less conservative than "balanced"
)
```

Or disable autotune and set explicit runtime config:

```python
runtime = mx8.RuntimeConfig(
    prefetch_batches=8,
    max_queue_batches=16,
)

loader = mx8.load(
    "s3://bucket/dataset/",
    max_ram_gb=24,
    autotune=False,
    runtime=runtime,
)
```
